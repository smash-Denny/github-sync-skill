# v1.0.6

## 核心变化

- feat(skill-bump): 修复step13输出顺序（违规时fatal立即退出，不打印"bump完成"）
- feat(skill-bump): 修复README版本历史顺序bug（改append为prepend到顶部）

## 文件变更

- `skill-bump.py`: 两处代码修复

## 变更说明

### bugfix-1: step13输出顺序
原逻辑：发现违规→打印警告→返回False→main()打印"bump完成"→exit 0
新逻辑：发现违规→step13调用fatal()立即sys.exit(1)→main()不打印"bump完成"

### bugfix-2: README版本历史顺序
原逻辑：`_readme_append_version_row`把新版本行追加到表格末尾
新逻辑：插入到表格顶部（所有现有数据行之前），保持最新版本在前的降序

## 根因

- bugfix-1根因：step13发现违规后只return False，未调用fatal()
- bugfix-2根因：`lines.insert(table_end+1)`插入到末尾，新版本出现在最底部而非最顶部
